```
This is not a frontend code issue — **S3 CORS is blocking the browser's PUT request.** The S3 bucket must be configured to allow browser PUT requests. There's nothing the frontend code can change to fix this.

Send this to your backend dev:

---

**Subject: S3 bucket needs CORS configuration for browser uploads**

The presigned URL flow is implemented correctly on our side (Step 1: get presigned URL, Step 2: PUT to S3). But the browser is getting **"Failed to fetch"** (a CORS block) when it tries to `PUT` directly to the S3 presigned URL.

**The S3 bucket needs a CORS policy added.** Please add this to the S3 bucket's CORS configuration:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<CORSConfiguration>
  <CORSRule>
    <AllowedOrigin>*</AllowedOrigin>
    <AllowedMethod>PUT</AllowedMethod>
    <AllowedMethod>GET</AllowedMethod>
    <AllowedHeader>*</AllowedHeader>
    <Max
```
